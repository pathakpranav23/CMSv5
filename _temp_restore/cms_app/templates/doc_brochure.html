Overview

CMSv5 is a modern, medium-aware College Management System that streamlines student data, fees, materials, announcements, and role-specific reports. Built for colleges operating Gujarati and English mediums across programs like B.Com, B.Sc, BCA, and BBA.

Why CMSv5

Medium-Smart Operations: Prevents mixing English/Gujarati cohorts via program-level policies and defaults.
Role-Tailored: Focused tools for Admin, Principal, Clerk, Faculty, and Student.
Safe Imports: Dry-run validation mode, clear alerts, and import audit logs.
Insightful Reports: Enrollment, fees, attendance, and materials summaries with CSV exports.
Localized UI: Gujarati and English with per-user preferences.

Feature Highlights

  • Students: Powerful search, Program/Semester/Medium filters, pagination, CSV export.
  • Subjects & Materials: Fast management hub, publish/review workflows, engagement summaries.
  • Fees: Structured heads, bulk import with Dry-run, receipts, UTR verification queue, bank details management.
  • Announcements: Targeted notice board with archive and dismissals.
  • Divisions: Planning forms, capacity management, rebalance tools.
  • Reports: Role-aware summaries, division utilization, absentees export, fees status, trends.
  • Audit & Health: Import logs with filters, system status cards for DB/Cache/Email/Storage.

Role Experiences

  • Admin: Global control, imports, reports, health checks, audit logs.
  • Principal: Division capacity vs enrolled, fees progress, medium splits.
  • Clerk: Bulk imports with Dry-run, verification queue, receipts.
  • Faculty: Attendance compliance, absentees CSV, materials publishing.
  • Student: Clean listings, targeted announcements, fee summaries.

Differentiators

  • Medium Policy embedded: B.Com supports English|Gujarati (default Gujarati); BBA/BCA/PG DMLT English-only; others Gujarati-only.
  • Validation-first workflows with Dry-run.
  • Localized Gujarati and English UI.
  • Comprehensive auditability.

Deployment

Recommended stack: Linux (Ubuntu) + Nginx + Gunicorn + Postgres + Redis. Health checks and config via Admin System Status.

Onboarding

Prepare programs.csv → Import subjects/students/fees with Dry-run → Set bank details → Publish announcements/materials → Assign faculty to subjects.

Call to Action

Start with a pilot program and semester, validate via Dry-run, enable reports and exports, then scale to all programs and mediums.
